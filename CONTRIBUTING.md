# Contributing to Elengal

Спасибо за интерес к проекту! 🙏

Elengal — это открытая платформа для исследований, и мы приветствуем вклад от сообщества.

---

## Философия контрибуций

Elengal построен на идеях **открытой науки** и **коллективного творчества**. Каждый вклад — это не просто код, а расширение возможностей платформы для всех.

---

## Лицензионное соглашение контрибьютора

**Важно:** Внося вклад в Elengal, вы соглашаетесь, что:

1. Ваш вклад будет лицензироваться под той же **двойной лицензией** (AGPLv3 или коммерческая)

2. Вы сохраняете авторские права на свой вклад, но предоставляете основному мейнтейнеру право использовать его в соответствии с условиями лицензий

3. Вы подтверждаете, что имеете право предоставить этот вклад (вы являетесь автором или имеете необходимые права)

```
By submitting a pull request, you agree to license your contribution 
under the same dual-licensing model (AGPLv3 or commercial).
You retain copyright but grant the project maintainer the right to 
use your contributions under these licenses.
```

## Процесс контрибуции

### 1. Fork и Clone

```bash
git clone https://github.com/YOUR_USERNAME/elengal.git
cd elengal
```

### 2. Создайте ветку

```bash
git checkout -b feature/your-feature-name
```

### 3. Внесите изменения

- Следуйте стилю кода проекта
- Добавьте тесты, если применимо
- Обновите документацию, если нужно

### 4. Проверьте

```bash
# Проверьте обучение
python train_elengal.py
```

### 5. Commit и Push

```bash
git add .
git commit -m "Add: описание изменений"
git push origin feature/your-feature-name
```

### 6. Pull Request

- Откройте PR в GitHub
- Опишите изменения
- Свяжите с Issue (если есть)
